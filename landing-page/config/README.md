# 🎯 Challenge Configuration Guide\n\n## Easy Challenge Updates\n\nTo update challenge content, dates, or details, simply edit the `challenges.ts` file. No need to touch component files!\n\n## 📝 How to Update Challenges\n\n### 1. Change Challenge Content\nEdit `/landing-page/config/challenges.ts`:\n\n```typescript\n// Example: Update a foundation exercise\n{\n  id: 1,\n  emoji: \"🤖\", // Change the emoji\n  title: \"LLM Basics\", // Update the title\n  description: \"Your new description here\", // Modify description\n  hasWorkshop: true, // Add/remove workshop\n  badge: \"New badge requirement\" // Update badge requirement\n}\n\n// Example: Update a main challenge\n{\n  id: 1,\n  emoji: \"📊\",\n  title: \"Dataset Task\",\n  description: \"New challenge description\",\n  date: \"Nov 1\", // Change the date\n  points: \"50-150\" // Update point range\n}\n```\n\n### 2. Update Dates\n```typescript\n// Change foundation launch date\nlaunchDate: \"Oct 15, 2025\",\n\n// Change main challenge start\nstartDate: \"Starting Nov 1\",\n\n// Update timeline milestones\ntimeline: {\n  milestones: [\n    { date: \"Oct 15\", label: \"Foundation Start\", description: \"...\" },\n    { date: \"Nov 1\", label: \"Main Challenge #1\", description: \"...\" },\n    // ...\n  ]\n}\n```\n\n### 3. Add New Challenges\n```typescript\n// Add a new foundation exercise\nexercises: [\n  // ... existing exercises\n  {\n    id: 8, // Next ID\n    emoji: \"🎯\",\n    title: \"New Exercise\",\n    description: \"Description of new exercise\",\n    hasWorkshop: false,\n    badge: \"Badge requirement\"\n  }\n]\n\n// Add a new main challenge\nchallenges: [\n  // ... existing challenges\n  {\n    id: 5, // Next ID\n    emoji: \"🚀\",\n    title: \"New Challenge\",\n    description: \"Description of new challenge\",\n    date: \"Dec 1\",\n    points: \"100-200\"\n  }\n]\n```\n\n### 4. Remove Challenges\nSimply delete the challenge object from the array or comment it out:\n\n```typescript\nexercises: [\n  // ... other exercises\n  // {\n  //   id: 7,\n  //   emoji: \"🚫\",\n  //   title: \"Prompt Jailbreaking\",\n  //   // ... temporarily disabled\n  // }\n]\n```\n\n## 🎨 Customization Options\n\n### Visual Elements\n- **Emojis**: Use any emoji for visual appeal\n- **Colors**: Workshop badges are green, date badges are OCT blue\n- **Layout**: Foundation exercises use 3-column grid, main challenges use 2-column\n\n### Content Fields\n- **Title**: Short, catchy challenge name\n- **Description**: One-sentence description of the challenge\n- **Badge**: Requirement to earn the badge (for foundation exercises)\n- **Points**: Point range or \"TBD\" (for main challenges)\n- **Date**: Release date for the challenge\n- **hasWorkshop**: Boolean to show/hide workshop badge\n\n## 🚀 Deployment\n\nAfter making changes:\n\n1. **Test locally**:\n   ```bash\n   cd landing-page\n   npm run dev\n   ```\n\n2. **Deploy**:\n   ```bash\n   git add .\n   git commit -m \"Update challenge roadmap\"\n   git push origin main\n   ```\n\n3. **GitHub Actions** will automatically deploy your changes!\n\n## 📅 Timeline Management\n\nThe timeline section automatically updates based on your milestone dates. Make sure to keep these in sync:\n\n- `foundationExercises.launchDate` → `timeline.milestones[0].date`\n- `mainChallenges.challenges[0].date` → `timeline.milestones[1].date`\n- Last challenge date → `timeline.milestones[2].date`\n\n## 🔧 Advanced Tips\n\n### Reordering Challenges\nChange the `id` values to reorder how they appear:\n\n```typescript\n// This will appear first\n{ id: 1, title: \"First Challenge\" },\n// This will appear second  \n{ id: 2, title: \"Second Challenge\" },\n```\n\n### Conditional Display\nYou can conditionally show/hide challenges based on dates:\n\n```typescript\n// In the component, filter by current date\nconst availableChallenges = challenges.filter(challenge => \n  new Date(challenge.date) <= new Date()\n)\n```\n\n### Dynamic Dates\nFor relative dates, you could compute them dynamically:\n\n```typescript\nconst today = new Date()\nconst foundationStart = new Date(today.getTime() + 7 * 24 * 60 * 60 * 1000) // 7 days from now\nlaunchDate: foundationStart.toLocaleDateString()\n```\n\n## 🎯 Best Practices\n\n- **Keep descriptions concise** (under 100 characters)\n- **Use consistent emoji themes** for similar challenge types\n- **Test changes locally** before deploying\n- **Update timeline milestones** when changing key dates\n- **Use \"TBD\"** for points that aren't finalized yet\n- **Add workshop badges** for exercises that include live sessions\n\n---\n\n## 🧩 Ultra-Modular Component Structure\n\n**The challenge system is now split into FIVE highly specialized components!**\n\n```\nlanding-page/\n├── components/\n│   ├── Hero.tsx                # Main hero section  \n│   ├── ChallengeRoadmap.tsx    # Foundation + main challenges\n│   ├── Timeline.tsx            # Interactive timeline\n│   └── README.md              # Component documentation\n├── config/\n│   ├── challenges.ts          # ← UPDATE THIS FILE\n│   └── README.md             # Instructions (this file)\n└── ...\n```\n\n**Benefits of ultra-modular structure:**\n- ✅ **Maximum flexibility** - Use any component anywhere\n- ✅ **Better organization** - Single responsibility per component\n- ✅ **Enhanced reusability** - Timeline on progress pages, challenges on overview pages\n- ✅ **Same easy updates** - Still just edit `challenges.ts`\n- ✅ **Independent testing** - Test each component separately\n\n**Usage examples:**\n```typescript\n// Use timeline on a status page\nimport Timeline from '../components/Timeline'\n\n// Use challenges on a overview page  \nimport ChallengeRoadmap from '../components/ChallengeRoadmap'\n\n// Use everything together (default)\nimport Hero from '../components/Hero'  // Includes both!\n```\n\n**Need help?** The challenge roadmap visualization will automatically update whenever you modify this configuration file! 🚀"
